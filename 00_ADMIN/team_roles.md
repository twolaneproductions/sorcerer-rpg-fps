# Team Roles and Responsibilities

## Overview
This document defines the core responsibilities and collaboration framework for a two-person indie game studio.

## Primary Responsibilities

### Person A — Creative Lead
- Narrative direction and story development
- Game design, systems flow, and player experience
- Dialogue writing, character voice, and script polish
- Playtesting coordination, feedback collection, and tuning

### Person B — Technical Lead
- Engine architecture and technology decisions
- Core systems, gameplay mechanics, and performance
- Level building, prototyping, and environment setup
- Implementation, integration, and technical problem solving

## Shared Responsibilities
- Establishing the game vision and high-level goals
- Iterating on design based on technical feasibility and player feedback
- Prioritizing scope and feature set for each milestone
- Maintaining project documentation and version control discipline
- Coordinating releases, builds, and testing cycles

## Decision Authority by Domain
- Narrative & Writing: Person A has final say, with technical input on implementation constraints
- Gameplay Design: Person A leads design direction; Person B vets technical feasibility
- Technical Systems: Person B has final authority on engine and systems choices
- Levels & Content: Shared decision-making, with Person B ensuring buildability and Person A ensuring player experience
- Schedule & Scope: Collaborative decision, with both leads agreeing on milestones and deliverables

## Conflict Resolution Process
1. Discuss the issue openly and identify the core disagreement.
2. Review the game vision, player goals, and current technical constraints.
3. Each person proposes a solution with pros, cons, and impact on the project.
4. Agree on the best course of action; prioritize the option that preserves momentum and quality.
5. If consensus is not reached, defer to the domain lead relevant to the disputed area, then reassess as needed.
